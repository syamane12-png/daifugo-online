# 大富豪オンライン対戦

友達4人(最大5人)がそれぞれ自分のスマホ・パソコンから同時に参加できる、大富豪のオンライン対戦です。
外部パッケージへの依存は一切ありません(Node.js標準機能のみで動作します)。`npm install` は不要です。

## ファイル構成

```
/
├ package.json
├ server.js       … HTTPサーバー・部屋(ルーム)管理・通信まわり
├ engine.js        … ゲームのルール・進行ロジックのみ(サーバー処理は含みません)
├ README.md
└ public/
    ├ index.html   … 画面のHTML・CSS
    └ client.js    … 画面の動き(サーバーと通信して表示を更新する部分)
```

`server.js` が `engine.js` を読み込んでゲームを進行させ、`public/` フォルダの中身をブラウザに配信します。

---

## 手順1: 動作確認(自分のパソコンで)

1. このフォルダをパソコンにダウンロードし、フォルダごと展開する
2. ターミナル(Macなら「ターミナル」、Windowsなら「コマンドプロンプト」)を開き、このフォルダに移動する
   ```
   cd daifugo-online
   ```
3. サーバーを起動する
   ```
   node server.js
   ```
4. 「大富豪オンラインサーバー起動: http://localhost:3000」と表示されたら成功
5. ブラウザで `http://localhost:3000` を開くと画面が表示されます

※ この状態では **自分のパソコンの中だけ** で動いています。友達が別の場所から参加するには、手順2でインターネットに公開する必要があります。

---

## 手順2: Renderに公開して、友達と遊べるようにする(5分程度)

Render(https://render.com)という無料で使えるホスティングサービスを使います。クレジットカード登録なしで始められます。

### 2-1. GitHubにファイルを置く(初めての場合)

Renderは「GitHubのリポジトリ」からコードを取得してデプロイする仕組みです。GitHubアカウントがない場合は https://github.com で無料登録してください。

1. GitHubにログインし、右上の「+」→「New repository」をクリック
2. リポジトリ名を決める(例: `daifugo-online`)、「Public」のままでOK、「Create repository」をクリック
3. 作成されたページの案内に従って、このフォルダの中身(`package.json` / `server.js` / `engine.js` / `public/` フォルダ一式)をアップロードする
   - GitHubの画面にある「uploading an existing file」というリンクからブラウザ上でドラッグ&ドロップでアップロードできます
   - `public` フォルダごとアップロードして構いません

### 2-2. Renderでサービスを作る

1. https://render.com にアクセスし、「Get Started」からアカウントを作成(GitHubアカウントでログインすると連携が楽です)
2. ダッシュボードで「New +」→「Web Service」をクリック
3. 「Build and deploy from a Git repository」を選び、先ほど作成したGitHubリポジトリを選択(連携の許可を求められたら許可する)
4. 設定画面で以下を入力:
   - **Name**: 好きな名前(例: `my-daifugo`)。これがURLの一部になります
   - **Region**: どこでも構いません(近い地域が多少速いです)
   - **Branch**: `main`(または `master`)
   - **Root Directory**: 空欄のまま
   - **Runtime**: `Node`
   - **Build Command**: 空欄のまま(または `echo "no build needed"`)
   - **Start Command**: `node server.js`
   - **Instance Type**: `Free` を選択
5. 一番下の「Create Web Service」をクリック
6. デプロイが始まります(2〜3分程度)。ログに `大富豪オンラインサーバー起動` と表示されれば成功です
7. 画面上部に `https://my-daifugo.onrender.com` のようなURLが表示されます。これが友達に共有するリンクです

### 2-3. 友達と遊ぶ

1. 発行されたURLを開く
2. 「ルームを作る」でルームコードが発行される
3. 「リンクをコピー」を押して、そのリンクを友達に送る(LINEなどで送るだけでOK)
4. 友達はリンクを開くだけで自動的にそのルームへの参加画面になります(名前を入力して「参加する」)
5. 2人以上集まったら、ホスト(最初に部屋を作った人)が「ゲームを始める」を押す
6. あとは画面の指示通り、手札を選んで「出す」「パス」で進行します

## 動作確認済みの範囲

以下の一連の流れを、実際にサーバーを起動してテストし、正常に動作することを確認しています。

1. 部屋作成
2. 友達が参加(複数人)
3. ホストがゲーム開始
4. カードが正しく配布される
5. カードを出す・パスする・7渡し/10捨て/縛り/革命などのルール判定
6. 上がり順・反則負けを含む勝敗判定
7. 次のラウンドへ進む
8. ロビーに戻ってゲーム終了

## 制限事項(正直な注意点)

- サーバーはメモリ上にルーム情報を持つだけです。**サーバーが再起動すると進行中のルームは消えます**。Renderの無料プランは一定時間アクセスがないとスリープし、次にアクセスがあった時に再起動するため、スリープを挟むと部屋の情報が消える点にご注意ください(遊ぶ直前に一度URLを開いて起動しておくと安心です)。
- 通信は約1.2秒ごとのポーリング(定期的な問い合わせ)方式です。WebSocketのような即時性はありませんが、カードゲームの操作感としては十分実用的です。
- 同時接続数やアクセス集中に対する特別な対策はしていません。友達内で使う分には十分ですが、不特定多数向けの公開は想定していません。
- 6時間操作がないルームは自動的に破棄されます。
- ルールの内容はローカル版・CPU対戦版と完全に共通です(`engine.js` を両方で共有しているため、縛り・激縛り・革命・禁止上がり・反則負けなどの判定は同一です)。
